    Notes:

    The Constraint Validation API: The constraint validation API lets you check inputs in a form for validity, and report back to the user if there's something wrong. An invalid form could be submitted by the user, but it would cause a validation error on the server when data is accepted from said user.

    Fetch API: The fetch API is a JavaScript API that allows you to make HTTP requests. It's a way to make requests to a server, and get back a response.

    Supplying request options The fetch API supports supplying request options. These options are passed in as an object, and can include the following properties:

    method: The HTTP method to use for the request. Defaults to "GET".

    headers: An object of HTTP headers to send along with the request.

    body: The body of the request. This is used for POST and PUT requests.

    mode: The mode of the request. This is used to determine the nature of the request (for example, whether the request should use CORS, or whether it should be sent with credentials). Defaults to "cors".

    credentials: The credentials of the request. This is used to determine whether the request should be sent with credentials. Defaults to "omit".



    Uploading JSON data: The fetch API supports uploading JSON data. This is done by supplying a body to the request, and setting the "Content-Type" header to "application/json".

    Uploading a file: Files are uploaded using an HTML input type="file" element, FormData() and the fetch API.

    Headers The fetch API supports supplying request headers. These headers are passed in as an object.
